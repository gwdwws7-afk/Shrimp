## 3. 游戏核心概念

### 3.1 游戏名称
**中文**: 深渊猎手：异种肃清  
**英文**: Abyss Hunter: Xenocide

### 3.2 世界观设定
> 公元2147年，深海钻探意外唤醒了沉睡亿万年的异种文明。这些生物机械混合体从海底深渊涌出，迅速占领了沿海城市。玩家扮演"深渊猎手"，装备着逆向工程制造的生物科技武器，独自深入异种巢穴，执行肃清任务。

### 3.3 核心玩法循环
```
进入关卡 → 割草战斗 → 收集资源 → 升级能力 → 挑战Boss → 解锁新区域
    ↑                                                    ↓
    └──────────────── 重复强化 ────────────────────────────┘
```

### 3.4 核心体验支柱

| 支柱 | 描述 | 实现方式 |
|------|------|----------|
| **爽快感** | 一刀下去敌人成片倒下 | 大范围攻击、夸张击退 |
| **成长感** | 角色越来越强大 | 技能树、装备升级 |
| **新鲜感** | 每次游戏都有新敌人 | AI实时生成变异模型 |
| **挑战性** | Boss战需要技巧 | 阶段式设计、弱点机制 |

### 5.2 大规模敌人群体系统

#### 敌人分级

| 类型 | 每波数量 | HP | 攻击力 | AI复杂度 | 特殊能力 |
|------|----------|----|--------|----------|----------|
| **杂兵(Grunt)** | 30-50 | 30 | 5 | ⭐ | 无 |
| **突击兵(Rusher)** | 10-15 | 50 | 10 | ⭐⭐ | 高速冲锋 |
| **重装兵(Tank)** | 5-8 | 150 | 15 | ⭐⭐ | 高防御,缓慢 |
| **精英(Elite)** | 3-5 | 300 | 25 | ⭐⭐⭐ | 闪避攻击 |
| **变异体(Mutant)** | 1-2 | 800 | 40 | ⭐⭐⭐⭐ | 范围技能 |
| **Boss** | 1 | 5000+ | 50+ | ⭐⭐⭐⭐⭐ | 阶段机制 |

#### 群体AI技术方案

**架构**: Unity DOTS (Data-Oriented Tech Stack)

```
敌人群体管理器
├── 对象池: 预生成200个敌人实体
├── 分区管理: 将地图划分为10x10网格
├── LOD AI:
│   ├── 近距离(<20m): 完整AI
│   ├── 中距离(20-50m): 简化AI
│   └── 远距离(>50m): 仅位置同步
├── 群体寻路: Flow Field算法
└── 同步机制: Burst编译 + Job System
```

**性能目标**:
- 同屏敌人: 50+ @ 60fps
- 敌人总数: 200 (对象池)
- 内存占用: < 500MB

## 6. AI生成内容管线

### 6.1 角色生成流程

```
阶段1: 概念生成
├── 输入: 敌人类型 + 关卡主题
└── 输出: 文字描述 + 关键词

阶段2: 3D模型生成
├── 工具: Meshy AI / Tripo3D / CSM
├── 参数:
│   ├── 多边形数: 5000-8000
│   ├── 贴图: 2K PBR (Albedo/Normal/Metallic/Roughness)
│   └── 格式: FBX
└── 输出: 基础模型

阶段3: 自动绑定
├── 工具: Mixamo / AccuRIG / Unity Animation Rigging
├── 骨骼数: 30-40根
└── 输出: 绑定好的模型

阶段4: 动画重定向
├── 源动画: Mixamo动作库(200+动作)
├── 目标: 生成的模型骨骼
└── 工具: Unity Animation Rigging

阶段5: 游戏集成
├── 导入Unity
├── 材质球设置
├── 预制体创建
└── 游戏参数配置
```

### 6.2 每关卡AI生成清单

| 场景模块 | Meshy | 15-20个 | 腐化程度 | 10分钟 |

## 7. 关卡设计

### 7.1 整体结构

**章节数**: 5章  
**每章节**: 3-4个关卡 + 1个Boss战

```
章节1: 浅海区域 (教程章节)
├── 关卡1-1: 海岸线清剿 (教学基础操作)
├── 关卡1-2: 废弃码头 (引入精英敌人)
├── 关卡1-3: 海底隧道 (环境挑战)
└── Boss: 深海巨蟹 (教学Boss机制)

章节2: 深海前哨
├── 关卡2-1: 研究站废墟
├── 关卡2-2: 异种孵化场
├── 关卡2-3: 能量核心室
└── Boss: 变异研究员

章节3: 深渊裂谷
...

章节5: 异种核心 (终章)
└── 最终Boss: 深渊之主
```

### 7.2 单关卡流程

```
起点 ──→ 战斗区A ──→ 探索区 ──→ 战斗区B ──→ 迷你Boss ──→ 终点
         ↑              ↓              ↑
      10-15杂兵    收集/升级      20+杂兵+精英
```

#### 标准关卡时长: 8-12分钟

### 7.3 程序生成要素

#### 房间模板库

| 房间类型 | 尺寸 | 功能 | 数量 |
|----------|------|------|------|
| 战斗房(小) | 15x15m | 15-20敌人 | 10种 |
| 战斗房(大) | 30x30m | 30-50敌人 | 8种 |
| 走廊 | 5x20m | 连接 | 15种 |
| 探索房 | 20x20m | 收集资源 | 5种 |
| 休息房 | 10x10m | 存档/回血 | 3种 |
| Boss房 | 40x40m | Boss战 | 每种Boss1个 |

#### 生成算法
- **基础**: Wave Function Collapse
- **变体**: 基于玩家等级调整敌人密度
- **特殊**: 随机环境hazard (毒雾/地刺/深渊)

## 10. 技术实现建议

### 10.1 核心系统技术选型

| 系统 | 技术方案 | 理由 |
|------|----------|------|
| 群体敌人 | Unity DOTS 1.0 | 高性能ECS架构 |
| 寻路 | Flow Field + NavMesh | 大规模群体最优 |
| 动画 | Animation Rigging | 程序化动画 |
| 特效 | Visual Effect Graph | GPU粒子 |
| UI | UI Toolkit | 高性能 |
| 存档 | JSON + AES | 简单安全 |
| 输入 | Input System | 跨平台 |

### 10.2 性能优化策略

```
敌人渲染:
├── GPU Instancing: 同模型批量渲染
├── LOD: 距离远降低模型精度
└── 遮挡剔除: 不可见不渲染

AI计算:
├── Burst编译: 数学运算加速
├── Job System: 多线程并行
└── LOD AI: 距离远简化逻辑

内存管理:
├── 对象池: 敌人/特效/音效
├── 动态加载: 关卡分块加载
└── 纹理流送: 按需加载贴图
```

## 12. 美术风格

### 12.1 视觉关键词

- **主色调**: 深蓝/暗紫/黑色
- **点缀色**: 生物发光 (青色/紫色)
- **材质**: 生物机械混合
- **氛围**: 深海恐惧 + 科技感
- **参考**: 死亡空间 + 深海迷航 + 战锤40K泰伦虫族

### 12.2 角色设计规范

#### 玩家角色
- 高科技潜水服 + 生物科技武器
- 头盔带发光目镜
- 动作风格: 力量感 + 流畅

#### 敌人设计
- 等级1-2: 原始海洋生物变异 (触手/鳞片)
- 等级3-4: 半机械化生物 (机械肢体)
- Boss: 巨大生物机械混合体

### 12.3 环境设计

#### 关卡主题色
| 章节 | 主色 | 辅助色 | 发光色 |
|------|------|--------|--------|
| 浅海 | 深蓝 | 灰白 | 浅蓝 |
| 深海 | 漆黑 | 暗紫 | 紫蓝 |
| 深渊 | 暗红 | 黑色 | 血红 |
| 核心 | 金色 | 黑色 | 白金 |

### 12.4 AI生成风格指南

```
一致性规则:
1. 所有生物必须有发光器官
2. 颜色限制在: 蓝/紫/青/红/黑
3. 避免过于写实的血腥
4. 机械部分要有腐蚀/寄生效果
5. 生物部分要有变异/增生特征
```
